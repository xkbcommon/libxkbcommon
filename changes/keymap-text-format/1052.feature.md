Added the following features for [modifier maps](@ref modmap-statement)
in the [keymap format v2](@ref ::XKB_KEYMAP_FORMAT_TEXT_V2):

<dl>
<dt>Modifiers mask expression</dt>
<dd>
```c
modifier_map Shift { … };
// New: numeric equivalent of the previous entry
modifier_map 0x001 { … };
// New: modifier mask expression
modifier_map Mod1+Mod2 { … };
// New: numeric equivalent of the previous entry
modifier_map 0x018 { … };
```
</dd>
<dt>[Extended real modifiers](@ref extended-real-mod-def)</dt>
<dd>
```c
// New: non-X11 core modifiers
modifier_map 0x100 { … };
// New: multiple non-X11 core modifiers
modifier_map 0xf00 { … };
// New: Mix of X11 core and non core modifiers
modifier_map Mod2+0x100 { … };
// New: Numeric equivalent of the previous entry
modifier_map 0x110 { … };
```
</dd>
<dt>Fallback to [keymap format v1](@ref ::XKB_KEYMAP_FORMAT_TEXT_V1)</dt>
<dd>
Due to backward compatibility with the X11 ecosystem, serializing to the
[keymap format v1](@ref ::XKB_KEYMAP_FORMAT_TEXT_V1) is subject to the following
restrictions:

- Non-[X11 core modifiers](@ref core-modifiers) are not supported.
- Multiple modifiers per key is not supported, unless using keysyms in the
  original `modifier_map` entry.
</dd>
</dl>
